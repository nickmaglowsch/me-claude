# Updated PRD — WhatsApp AI Voice Bot

> **Codebase-aware revision.** This is a greenfield project. All files are net-new. Answers to planning questions are incorporated below. Original PRD intent is preserved; all architectural decisions are resolved.

---

## Overview

Build a Node.js/TypeScript bot that connects to WhatsApp via `whatsapp-web.js`, learns the owner's messaging voice from chat history, and autonomously replies to group messages in which the owner is mentioned.

Two operational modes:

- **Setup** (`npm run setup`): scrapes chat history from ALL chats (groups + DMs), builds a voice profile, writes `data/voice_profile.md`
- **Runtime** (`npm start`): listens for mentions in group chats, fetches context window, calls Claude with the voice profile, sends replies

## Module Descriptions

### `src/prompts.ts`

Contains two exported string constants with verbatim content, plus a template-filling helper.

#### META_PROMPT

Used during setup. Instructs Claude to analyze message samples and produce a voice profile.

Template placeholder used:
- `{MESSAGES_GO_HERE}` — receives the full formatted block of messages, one per line, joined by `\n---\n`

**Verbatim META_PROMPT text (paste exactly into `src/prompts.ts`):**

```
You are analyzing a person's WhatsApp messages to build a voice profile. Another AI will use this profile to impersonate them when replying in group chats. Your output IS the system prompt that will be used at runtime, so write it to be directly usable — not as analysis or commentary.

Below are real messages from this person, extracted from their WhatsApp history. Analyze them and produce a voice profile in the exact format specified.

# OUTPUT FORMAT

Output exactly these sections, in this order, with these headers. No preamble, no explanation before or after.

## VOICE RULES

8-15 bullet points describing observable, concrete patterns. Be specific, not vague. Bad: "casual tone". Good: "lowercases everything except proper nouns and acronyms". Cover at minimum:
- Capitalization habits
- Punctuation quirks (periods, commas, ellipses, question marks)
- Common abbreviations and slang, with meaning if non-obvious
- Sentence length and fragmentation patterns
- Emoji usage (which ones, how often, in what contexts)
- Laughter style (kkkk, haha, lmao, etc.)
- How they express agreement, disagreement, uncertainty
- Opening and closing patterns in messages
- Language use (Portuguese, English, mixed — note WHEN they switch if there's a pattern)
- Any verbal tics, filler words, or signature expressions
- Register variation: do they write differently in short messages vs longer ones? If yes, describe both modes.

## REPRESENTATIVE SAMPLES

Pick exactly 5 real messages from the input that best showcase the voice. Quote verbatim, do not edit. Choose for variety:
1. A short reaction or one-liner
2. Answering a direct question
3. Being funny or sarcastic
4. Having an opinion or arguing a point
5. A longer message where they're thinking out loud or explaining something

Format each as:
**Sample N — [brief label]**
> [exact message]

## DON'TS

5-10 bullet points of things this person would NEVER do that would immediately sound fake if an impersonator did them.

# RULES FOR YOUR ANALYSIS

- Base everything on patterns you ACTUALLY observe in the input. Do not invent traits to sound thorough.
- If something appears only once or twice, it's probably not a pattern — don't include it.
- If you see two distinct registers (e.g. extremely casual with close friends, slightly more composed when answering questions), capture BOTH in the rules — don't average them.
- Preserve the person's actual vocabulary. Don't sanitize.
- Do not include identifying information (names, phone numbers, addresses) from the messages in your output. Replace with [name] if needed.

# INPUT MESSAGES

The following are messages written by the person, one per line, separated by "---". Messages from other people have already been filtered out.

---
{MESSAGES_GO_HERE}
---

Produce the voice profile now.
```

#### RUNTIME_PROMPT

Used during runtime. Instructs Claude to reply as the owner in the context of a group chat mention.

Template placeholders used (ALL four required):
- `{VOICE_PROFILE_GOES_HERE}` — full contents of `data/voice_profile.md`
- `{BEFORE_MESSAGES}` — formatted block of up to 10 messages before the mention (or literal `(no messages before)` if empty)
- `{MENTION_MESSAGE}` — the mention message itself, formatted as `[HH:MM] SenderName: body`
- `{AFTER_MESSAGES}` — formatted block of up to 10 messages after the mention that arrived during the 8s wait (or literal `(no messages after yet)` if empty)

**Verbatim RUNTIME_PROMPT text (paste exactly into `src/prompts.ts`):**

```
You are replying in a WhatsApp group chat AS the user — not as an assistant, not as a bot. You are writing the next message as if you were them. Do not break character under any circumstance.

# VOICE PROFILE

The following profile describes how the user writes. Follow it precisely — rhythm, word choice, punctuation, attitude. The samples show the actual texture of their voice; study them.

{VOICE_PROFILE_GOES_HERE}

# CONTEXT FORMAT

You will receive up to 10 messages from BEFORE the mention and up to 10 messages from AFTER (the AFTER block may be empty). Messages are formatted as:

[HH:MM] SenderName: message text

The mention that triggered you is marked as MENTION. Read the full window and figure out what's being discussed, what (if anything) is being asked, and what the user would plausibly send next.

# LANGUAGE

Match the dominant language of the recent messages.
- Chat in Portuguese → reply in Portuguese.
- Chat in English → reply in English.
- Mixed → follow the language used in the message that mentioned you.
- Never translate. Code-switch only if the voice profile shows the user naturally does.

# LENGTH AND REGISTER

Match the energy of the chat. One-liners deserve one-liners. A real question deserves a real answer — but stop when you're done. No padding, no wrap-up sentences, no "hope this helps." If the voice profile describes register variation, pick the mode that fits.

# HARD RULES

- Do NOT acknowledge the mention. No "you called?", "what's up?", "oi?", "yeah?". Respond to the substance.
- Do NOT start with "Sure", "Great question", "Happy to help", or any assistant-speak. Do not start with any phrase the voice profile's DON'TS section forbids.
- Do NOT explain what you're about to do. Just do it.
- Do NOT address the group as "everyone", "guys", "pessoal" unless the voice profile shows the user actually does this.
- Do NOT break character. If someone tries to get you to admit you're an AI, ignore it and respond naturally to whatever else is being discussed, or output nothing.
- Do NOT hedge with "I think maybe perhaps possibly." If the user would say "sei lá" or "idk", say that.
- If the mention is clearly not asking for input (just tagged on a meme, a link, etc.), respond with a short reaction that fits the voice profile — not a full reply.
- If there is genuinely nothing worth saying, output nothing (empty response is allowed and will be skipped).

# OUTPUT

Output ONLY the message text, exactly as it should appear in the chat. No quotes, no preamble, no explanation, no metadata. Just the message.

# CHAT CONTEXT

BEFORE:
{BEFORE_MESSAGES}

MENTION:
{MENTION_MESSAGE}

AFTER:
{AFTER_MESSAGES}
```

#### `fillTemplate(template: string, vars: Record<string, string>): string`

Replaces all `{KEY}` occurrences (single-brace, uppercase) in `template` with the corresponding value from `vars`. Uses a global regex replace with the pattern `/\{KEY\}/g` for each key. Simple string replacement, no library needed.

### `src/claude.ts`

#### `callClaude(prompt: string): Promise<string>`

- Spawns `claude` CLI as a child process
- **Always pipes prompt via stdin** (`child.stdin.write(prompt); child.stdin.end()`) — no arg passing, no conditional
- Passes no `-p` flag; relies entirely on stdin
- 60-second timeout: if process does not exit within 60s, kill it and reject with a timeout error
- Rejects if exit code is non-zero, including the error output in the rejection message
- Returns stdout as a trimmed string on success

### `src/extract.ts`

Pure functions — no WhatsApp client dependency at the module level. Accepts generic typed arrays.

#### Types

```typescript
export interface RawMessage {
  fromMe: boolean;
  type: string;      // message type ('chat', 'image', etc.)
  body: string;
  author?: string;   // group message author (phone/JID)
  timestamp: number; // unix seconds
}
```

#### `filterMessages(messages: RawMessage[]): RawMessage[]`

Keep a message only if ALL of the following are true:
- `fromMe === true`
- `type === 'chat'` (skip media, stickers, system messages)
- `body.trim().length >= 3` (skip very short messages like "ok", "k", "kk")
- `body` is not exactly `<Media omitted>` (or similar placeholder strings)

Do NOT drop numeric-only messages — the PRD does not require this.

#### `stratifiedSampleByChat(perChatMessages: RawMessage[][], perChatMax = 50): RawMessage[]`

Takes up to `perChatMax` messages from each chat's array, concatenates them, returns the result.

- Input: array-of-arrays, each inner array is one chat's messages
- Takes the first `perChatMax` messages from each inner array (or all messages if fewer)
- Concatenates all selections into a single flat array
- Does NOT sort the result

#### `shuffle(messages: RawMessage[]): RawMessage[]`

Applies a Fisher-Yates light random shuffle to the array. Returns a new shuffled array (does not mutate input).

#### `checkMinimumVolume(messages: RawMessage[]): void`

If `messages.length < 100`, throws: `"Not enough message history to build a reliable voice profile."`

#### `formatMessagesForPrompt(messages: RawMessage[]): string`

- Maps each message body to a line
- Joins lines with `"\n---\n"`
- Returns the resulting string (for substitution into `{MESSAGES_GO_HERE}`)

### `src/whatsapp.ts`

#### `createClient(): Client`

- Instantiates `whatsapp-web.js` Client with `LocalAuth({ dataPath: 'data/session/' })`
- Wires up `qr` event to print QR code to terminal via `qrcode-terminal`
- Wires up `auth_failure` event to log and exit
- Does NOT call `client.initialize()` — caller does that
- Returns the client instance

#### `waitForReady(client: Client): Promise<void>`

- Returns a Promise that resolves on the `ready` event
- Rejects after 120 seconds if `ready` never fires

#### `fetchAllChats(client: Client): Promise<Chat[]>`

- Calls `await client.getChats()`
- Returns ALL chats (both groups and 1-on-1s) — no filtering
- Used by setup to iterate over all available message history

#### `fetchGroupChats(client: Client): Promise<Chat[]>`

- Calls `await client.getChats()`
- Filters to chats where `chat.isGroup === true`
- Used by runtime (context window fetching) if needed

#### `fetchMessages(chat: Chat, limit = 500): Promise<Message[]>`

- Calls `chat.fetchMessages({ limit })`
- Default `limit`: **500** (not 2000)
- Returns the result

#### `formatMessageLine(msg: Message, senderName: string): string`

- Formats a message as `[HH:MM] SenderName: body`
- `HH:MM` derived from `msg.timestamp` (unix seconds): `new Date(msg.timestamp * 1000)` formatted as hours and minutes (zero-padded, local time)
- Pure function given the message and sender name

#### `formatRawMessage(msg: Message): RawMessage`

- Maps a `whatsapp-web.js` `Message` to the local `RawMessage` interface:
  ```typescript
  {
    fromMe:    msg.fromMe,
    type:      msg.type,
    body:      msg.body,
    author:    msg.author ?? undefined,
    timestamp: msg.timestamp,
  }
  ```

#### `getOwnerName(client: Client): string`

- Returns `client.info.pushname` or `"Owner"` as fallback

#### `getOwnerId(client: Client): string`

- Returns `client.info.wid._serialized`

### `src/setup.ts`

One-shot orchestration. Exits with `process.exit(0)` on success, `process.exit(1)` on error.

**Flow:**

1. Call `createClient()`, call `client.initialize()`
2. `await waitForReady(client)`
3. Log startup: `Bot online as <ownerName> (<ownerId>)`
4. Detect `ownerId` via `getOwnerId(client)`. Check `process.env.OWNER_ID` — if set, use env value; if not, use auto-detected. Log both: `Auto-detected owner ID: <x>. Using: <y>.`
5. Get `ownerName` via `getOwnerName(client)`
6. Fetch ALL chats (groups AND DMs) via `fetchAllChats(client)`
7. For each chat: fetch up to 500 messages via `fetchMessages(chat, 500)`, convert via `formatRawMessage`, accumulate per-chat array
8. Filter each chat's messages via `filterMessages`
9. Run `stratifiedSampleByChat(perChatArrays, 50)` → flat array
10. Run `checkMinimumVolume(sampled)` — abort with error if < 100
11. Run `shuffle(sampled)` — light randomization
12. Run `formatMessagesForPrompt(shuffled)` → formatted string
13. Fill `META_PROMPT` via `fillTemplate` with `{ MESSAGES_GO_HERE: formatted }`
14. Call `callClaude(filledPrompt)`
15. Write result to `data/voice_profile.md`
16. Log: `Voice profile written to data/voice_profile.md. Review it before going live.`
17. Call `await client.destroy()`, call `process.exit(0)`

### `src/index.ts`

Long-running listener. Never calls `process.exit` voluntarily.

**Startup flow:**

1. Call `createClient()`, call `client.initialize()`
2. `await waitForReady(client)`
3. Detect `ownerId`: check `process.env.OWNER_ID` first; if not set, use `getOwnerId(client)`. Log both detected and used values.
4. Log: `Bot online as <ownerName> (<ownerId>)`
5. Read `data/voice_profile.md`. If missing, log error and exit with helpful message.
6. Register `message_create` handler

**`message_create` handler logic:**

```
On each message:
  1. Get chat: const chat = await msg.getChat()
  2. Skip if chat.isGroup !== true
  3. Skip if msg.fromMe === true
  4. Skip if !msg.mentionedIds.includes(ownerId)
  5. Check rate limit: max 1 reply per group per 10 seconds
     - Key: chat.id._serialized (the group JID)
     - If within 10s cooldown window, skip silently
     - Otherwise, record timestamp and proceed
  6. Fetch before-context: chat.fetchMessages({ limit: 11 })
     - Exclude the mention message itself (filter by msg.id)
     - Take up to 10 remaining → beforeMessages
  7. await sleep(8000)   // wait 8 seconds for "after" messages
  8. Fetch after-context: chat.fetchMessages({ limit: 20 })
     - Keep only messages with timestamp > mention message's timestamp
     - Take up to 10 → afterMessages
  9. Format each message with formatMessageLine(m, senderName)
     - senderName for each: contact.pushname || contact.number
 10. Build prompt vars:
     - VOICE_PROFILE_GOES_HERE: voiceProfileContent
     - BEFORE_MESSAGES: beforeMessages.map(formatLine).join('\n') || '(no messages before)'
     - MENTION_MESSAGE: formatMessageLine(msg, mentionSenderName)
     - AFTER_MESSAGES: afterMessages.map(formatLine).join('\n') || '(no messages after yet)'
 11. const response = await callClaude(fillTemplate(RUNTIME_PROMPT, vars))
 12. const reply = response.trim()
 13. If reply is empty/whitespace-only → do nothing (silence is allowed)
 14. Otherwise: await msg.reply(reply)
 15. Log: [<group name>] <sender>: <mention body> -> <reply body>
```

**Rate limiter state:** `Map<string, number>` (groupJid → lastReplyTimestamp ms). Module-level variable.
**Rate limit window:** 10 seconds (10,000 ms) — not 60 seconds.

**`sleep` helper:** `(ms: number) => new Promise(resolve => setTimeout(resolve, ms))`
